### Task 3: Slots

**Files:** `packages/app.js/src/app.ts`; Test (create): `packages/app.js/tests/slots.test.ts`.

- [ ] **Parent side, at wiring (BEFORE data-for extraction):** the combo-ban sweep (`data-slot` + `data-show-if`/`data-for` on one element → loud error, `data-slot` ignored → default routing; empty `data-slot=""` → loud error); the in-item unconditional error (wrapper content on any `data-component` inside a `data-for` item).
- [ ] **Child side, FIRST sweep of the child's template:** the slot record — for each `<slot>`: name (default = unnamed), position marker, fallback childNodes extracted UNWIRED into a held fragment; duplicate names (incl. two defaults) → loud error; `<slot>` inside the child's own `data-for` → loud error + removal (via `closest('[data-for]')`, before extraction); directives on the `<slot>` element → loud error; nested `<slot>` inside fallback → loud error; `<slot>` in the ROOT component's template → loud error (no parent).
- [ ] **Distribution, in `#loadComponent`'s continuation** — after the destroyed gate, BEFORE the child fragment appends: collect wrapper childNodes (purely parent content at that point); route top-level meaningful nodes by `data-slot` (elements) / default (everything else, order preserved); consume the slot RECORD (never a DOM query — detached slots still fill via `replaceWith` inside their subtree): filled bucket → nodes replace the slot, held fallback fragment dropped unwired; empty bucket → wire the fallback NOW via the Task-2 seam (child scope) and unwrap it in place; then the errors: `data-slot` name with no matching slot; meaningful default-bucket content with no default slot; ANY meaningful wrapper content on a slotless template (content REMOVED — the retirement). Meaningful = non-whitespace text or non-comment nodes (the SFC-file ignorable rule). Fallback-nested component mounts fold into the promise chain `ready` awaits.
- [ ] **Tests — the spec §D list in full** (each RED where it pins new behavior): default projection incl. duplicate-name buckets and order; named routing (top-level only; nested inert; on a nested `data-component` wrapper); fallback lazy proof (a fallback SFC's template is NEVER fetched when the bucket fills — spy on fetch); parent-scope proof through `settle`; hidden-at-migration default-bucket content (a projected `data-show-if` element hidden pre-mount, shown post — and the named case via a `data-slot` WRAPPER); a projected `data-for` reconciling inside a slot via a routed wrapper; slot-under-child-show-if detach/reattach incl. slot-detached-at-distribution (an include sibling forcing the early drain); every §A.6 error incl. removal assertions; whitespace-only wrapper content is NOT an error; template-only includes with content keep today's semantics (regression).
